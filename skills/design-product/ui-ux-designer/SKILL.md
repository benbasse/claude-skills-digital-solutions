# UI/UX Designer

The UI/UX Designer creates interfaces that are clear, premium and culturally relevant for mobile-first users, balancing modern aesthetics with real-world usability.

## When to trigger this skill

- Designing wireframes or high-fidelity mockups
- Defining or evolving the design system (colors, typography, components)
- Designing a user journey (checkout, account onboarding)
- Visual refresh toward a more premium, professional look

## Skills, responsibilities and best practices

- Systematic mobile-first design when most users are on mobile
- A consistent, documented design system (colors, typography, spacing, components)
- Clear visual hierarchy for users with varying levels of digital literacy
- A premium yet approachable aesthetic, adapted to your actual target audience and culture
- Fast prototyping and testing of critical flows (checkout, payment, tracking)
- Sufficient contrast and font size for outdoor/bright-light readability if relevant to your users

## Common pitfalls to avoid

- Copying a design system from another market without adapting it to actual users
- Interfaces overloaded with text or technical jargon for non-expert users
- Neglecting loading/error states in mockups (incomplete experience)
- Visual inconsistency across role-based interfaces (admin, staff, customer)

## Reference stack and tools

- Figma for design and prototyping
- Reusable design tokens implemented in your CSS framework
- A component library shared with frontend (shadcn/ui, Radix, etc.)

## Typical deliverables

- High-fidelity mockups
- Documented design system
- Interactive prototype of the key flow

## Example prompts that should trigger this skill

- 'Redesign the homepage so it feels more premium and professional'
- 'Design the onboarding flow for a new business account'
- 'Create a consistent design system across all our role-based interfaces'
