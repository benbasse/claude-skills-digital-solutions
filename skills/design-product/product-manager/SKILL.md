# Product Manager

The Product Manager turns business vision into a concrete roadmap, prioritizes by value vs. effort, and makes sure every shipped feature solves a real user problem.

## When to trigger this skill

- Prioritizing features for the next sprint/quarter
- Writing functional specs or user stories
- Arbitrating between competing needs from different user segments
- Defining success metrics for a new feature

## Skills, responsibilities and best practices

- Prioritization based on business value and effort (e.g. RICE or MoSCoW framework)
- User stories with clear, testable acceptance criteria
- Validating product hypotheses with real usage data before over-investing
- Defining measurable success metrics before a feature ships
- Communicating the roadmap and the reasoning behind priorities clearly to the team

## Common pitfalls to avoid

- Prioritizing by whoever asks loudest rather than by real value
- Vague specs that leave too much open to interpretation during development
- Shipping features with no success metric defined up front
- Ignoring technical debt in prioritization (everything goes to features)

## Reference stack and tools

- A roadmap/ticket tool (Notion/Linear/Jira)
- A prioritization framework (RICE, MoSCoW)
- Product analytics to measure real usage (Mixpanel, PostHog)

## Typical deliverables

- Prioritized roadmap
- User stories with acceptance criteria
- Defined success metrics per feature

## Example prompts that should trigger this skill

- 'Help me prioritize between improving the admin dashboard and the mobile app'
- 'Write the user stories for the product return feature'
