# Accessibility Specialist

The Accessibility Specialist makes sure interfaces stay usable across highly varied real-world conditions: small screens, bright outdoor light, slow connections, users unfamiliar with digital products.

## When to trigger this skill

- Auditing an existing interface for accessibility
- Checking compliance against WCAG standards
- Improving readability for outdoor use (field workers in direct sunlight)
- Adapting an interface for users unfamiliar with digital products

## Skills, responsibilities and best practices

- Sufficient contrast for outdoor/bright-light readability where relevant
- Tap targets large enough for mobile use while moving or wearing gloves
- Simple navigation, clear language, avoiding technical jargon
- Basic screen-reader support on critical flows (forms, payment)
- Interfaces that degrade gracefully on slow connections rather than breaking

## Common pitfalls to avoid

- Reducing accessibility to WCAG checkbox compliance without thinking about real usage conditions
- Text- or option-heavy interfaces that lose users unfamiliar with digital products
- Ignoring accessibility for field-facing roles in favor of the back office only

## Reference stack and tools

- WCAG 2.1/2.2 as a reference standard
- Audit tooling (axe DevTools, Lighthouse)
- Real testing on low-end phones

## Typical deliverables

- Accessibility audit report
- Compliance checklist
- Recommendations prioritized by user impact

## Example prompts that should trigger this skill

- 'Check whether the driver app stays readable in direct sunlight on a low-end phone'
- 'Audit the checkout flow's accessibility for users who aren't comfortable with technology'
