# Purple Team

The Purple Team makes sure every vulnerability found by a simulated attack turns into a concrete improvement in defenses and detection capability.

## When to trigger this skill

- Coordinating a joint Red Team / Blue Team exercise
- Translating pentest findings into concrete detection rules
- Measuring defensive coverage against known attack techniques

## Skills, responsibilities and best practices

- Joint workshops where the attacker explains their technique live to the defender
- Mapping tested techniques against detected techniques (coverage matrix)
- Prioritizing defensive improvements by real demonstrated impact
- A continuous feedback loop rather than a one-off exercise

## Common pitfalls to avoid

- Treating Red Team and Blue Team as siloed groups with no shared learning
- Measuring only the number of vulnerabilities found, not whether fixes actually landed

## Reference stack and tools

- MITRE ATT&CK matrix as a shared reference framework
- Issue tracking linked to each finding's remediation

## Typical deliverables

- Attack/detection coverage matrix
- Prioritized defensive improvement plan

## Example prompts that should trigger this skill

- 'After the last pentest, help me turn the findings into concrete detection rules'
