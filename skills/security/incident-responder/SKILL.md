# Incident Responder / Forensics

The Incident Responder quickly contains a confirmed security incident, preserves evidence, determines root cause, and documents the incident to prevent recurrence.

## When to trigger this skill

- A confirmed security incident (account compromise, unauthorized access, data leak)
- Forensic analysis following confirmed suspicious activity
- Writing an incident report for leadership or relevant authorities
- Determining the root cause of a compromise

## Skills, responsibilities and best practices

- Structured methodology: identification -> containment -> eradication -> recovery -> lessons learned
- Preserving evidence before any remediation action (logs, snapshots)
- Controlled, factual communication during the incident (avoid panic or over-communication)
- A precise timeline of events for the final report
- Notifying affected parties if personal data is involved

## Common pitfalls to avoid

- Fixing immediately without preserving evidence (root cause becomes unknowable)
- Rushing public communication before the facts are confirmed
- No formal post-incident report -> no organizational learning

## Reference stack and tools

- Centralized logs and database snapshots
- Log analysis and event-timeline tooling

## Typical deliverables

- Full incident report (timeline, root cause, actions taken)
- Remediation plan
- Communication to affected parties if required

## Example prompts that should trigger this skill

- 'An admin account looks compromised, help me contain and investigate it'
- 'Write the post-incident report for yesterday's data leak'
